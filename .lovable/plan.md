

## «Место рядом» — Полный MVP

### Фаза 1: Фундамент
- **Настройка Supabase (Lovable Cloud)**: база данных, аутентификация, хранилище файлов
- **Схема БД**: таблицы `lots`, `client_applications`, `host_applications`, `verification_documents`, `site_documents` с RLS-политиками и ролями (admin)
- **Дизайн-система**: зелёно-бирюзовая палитра, светлый фон, крупная типографика, mobile-first. Логотип из вложения
- **Навигация**: хедер с логотипом и ссылками (Снять место, Сдать место, Как это работает, FAQ), футер с юридическими документами

### Фаза 2: Клиентские страницы
- **Главная**: Hero-блок с заголовком и двумя CTA, блок категорий (Шины/Велосипеды/Другое), «Как это работает» (для клиента и хоста), преимущества, примеры лотов, FAQ, контакты
- **Страница «Снять место»**: список лотов с карточками, фильтры (тип товара, режим доступа), Яндекс.Карта с пинами, синхронизация карты и списка, мини-карточки по клику
- **Карточка лота**: фото, цена, категория, адрес, режим доступа, описание, правила, Яндекс.Карта, форма заявки. Плашка для лотов вне Мытищ. Дисклеймер для категории «Другое»
- **Страница «Сдать место»**: лендинг для хостов с оффером, объяснением, формой подачи (имя, телефон, адрес с автоподсказкой Яндекса, тип места, категория, режим доступа, фото), FAQ для хостов

### Фаза 3: Верификация и заявки
- **Верификация телефона клиента** (обязательна перед заявкой)
- **Верификация хоста**: загрузка документа (паспорт/ВУ), статусы проверки
- **Отклик клиента на лот**: форма заявки (имя, телефон, категория, дата, комментарий) → сохранение в БД → передача хосту
- **Гибкая архитектура для документной верификации клиента** (включается/отключается)

### Фаза 4: Админка
- **Раздел заявок клиентов**: таблица со всеми данными, статусы (Новая → Передана хосту → Завершено/Отказ)
- **Раздел заявок хостов**: таблица с данными, фото, статусы верификации
- **Раздел лотов**: CRUD (создание, редактирование, публикация, архивация), управление статусами (Черновик → Опубликовано)
- **Раздел документов сайта**: управление юридическими документами, версионирование, включение/отключение

### Фаза 5: Интеграции
- **Яндекс.Карты**: карта на странице «Снять место» и в карточке лота, автоподсказки адресов
- **Хранилище файлов**: Supabase Storage для фото лотов и документов верификации
- **Email-уведомления** (архитектурно): при новой заявке — уведомление хосту

### Технические детали
- React + TypeScript + Tailwind, mobile-first
- Supabase: Auth, Database (PostgreSQL), Storage, Edge Functions
- Яндекс.Карты JS API
- Контент (тексты, FAQ, дисклеймеры) управляется через БД/админку, не зашит в код
